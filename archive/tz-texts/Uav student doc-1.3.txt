Неделя 1. Архитектура полётного контроллера
Цель недели

“Почему не работает GPS?”

1. Что изучаем
Основные компоненты FC

UART порты
SPI / I2C шины
Timer ресурсы
PWM / DShot outputs
ADC (ток / напряжение)
Важные понятия
что такое UART и почему их всегда не хватает
почему SPI быстрее I2C
почему не все пины взаимозаменяемы
что такое resource mapping (актуально для Betaflight)
2. Как это работает внутри
FC — это по сути:
Plain text
MCU
 ├── UART → RX / GPS / Telemetry / Companion
 ├── SPI → IMU / Flash
 ├── I2C → Compass / Baro
 ├── Timers → Motors / Servos
 └── ADC → Battery / Current
В любой прошивке:
данные идут с сенсоров → в контроллер → в моторы
а ограничения идут от железа, не от прошивки

2. Как это работает
Главный принцип
Plain text
IMU (gyro+accel) = стабилизация
GPS = позиция
Baro = высота
Compass = направление
Если IMU плохой → дрон нестабилен
Если GPS плохой → дрон тупит в навигации
Если компас плохой → улетит не туда
Если баро шумит → прыгает по высоте
