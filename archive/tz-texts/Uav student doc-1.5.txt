2. Логика настройки (запомни порядок)
Если ты делаешь в другом порядке — ты сам себе враг.
Plain text
Прошивка → Ports → Receiver → Configuration → Motors → Modes → OSD
3. Практика
Задание 1. Прошивка FC
выбрать target
выбрать версию
Full chip erase (если нужно)
прошить
Задание 2. Настройка портов
Plain text
UART1 → RX

UART3 → VTX / Telemetry
Задание 3. Receiver
Проверить:
двигаются ли каналы

Задание 4. Motors

Минимум:
ARM

BEEPER

не тот target

Неделя 9. CLI и дисциплина конфигурации
Цель недели
Перестать зависеть от GUI и начать контролировать конфиг как инженер.
1. Что изучаем
Основные команды CLI
Bash
diff
dump
get
set
save
👉 CLI docs:
https://betaflight.com/docs/development/Cli⁠�
2. Логика работы
diff → только изменения
dump → полный конфиг
get → посмотреть параметр
set → изменить
save → применить
3. Практика
Задание 1. Сделать backup
Bash
diff all
Сохранить в файл.
Задание 2. Изменить параметр
Bash
set yaw_rate = 80
save
Задание 3. Восстановление
сбросить конфиг
применить сохранённый diff
4. Типовые ошибки
не делать backup
копировать чужой diff без понимания
забывать save
5. Мини-проект
Создать:
“Мой базовый CLI профиль”

Неделя 10. PID и фильтры
Цель недели
Понять, почему дрон летает так, как летает, а не “подкрутил и вроде ок”.
1. Что изучаем
Основные компоненты
PID
Feedforward
Dynamic notch
RPM filtering
Dynamic idle
2. Как это работает
Упрощённо:
Plain text
PID = исправление ошибки
Feedforward = предсказание движения
Filters = борьба с шумом
3. Практика
Задание 1. Базовый тюн
поднять P
проверить осцилляции
Задание 2. Настройка фильтров
включить RPM filter
настроить dynamic notch
Задание 3. Проверка
быстрые стики
резкие остановки
yaw
4. Типовые ошибки
отключить фильтры “чтобы было быстрее”
завысить PID
игнорировать вибрации
